● VERIFIED

OmniGupt is a client-side modern cryptography laboratory and encryption analysis toolkit designed for developers, learners, and security analysts.

Privacy Model

All cryptographic operations are performed client-side in the browser. OmniGupt does not intentionally persist secret keys, passphrases, or plaintext messages. The required CryptoJS library is loaded securely via a public CDN. Operational audit metadata is saved locally to browser storage, and optionally logged to server.py if the local Python server is running.

Legacy Algorithms Policy

Legacy algorithms (DES, Triple DES, Rabbit) are included strictly for educational analysis and retro-compatibility. They are not recommended for protecting modern sensitive data. AES-256-GCM is recommended for all new encryption operations due to its modern authenticated encryption features.

Security Limitations

OmniGupt is an educational and developer cryptography toolkit. Users should evaluate security requirements before using browser-based tools for highly sensitive production data. The tool is provided without warranties, compliance certificates, or professional cryptographic audits.
